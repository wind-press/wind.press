## What‚Äôs New in WindPress?

### ‚ú® Always Up-to-Date

Tailwind‚Äôs minor and patch updates are backward-compatible. This means you can safely update to the latest version without worrying about compatibility issues.

In the past, Winden gave users the option to select a specific minor or patch version of Tailwind CSS. While well-intentioned, this added unnecessary complexity. WindPress simplifies this by always using the latest stable version of Tailwind CSS. This ensures you automatically benefit from the newest improvements without extra configuration. ### ‚õìÔ∏è A Unified Engine

<!-- Winden‚Äôs legacy architecture involved different engines for development and production, causing inconsistencies and bugs. WindPress fixes this with a unified engine that works seamlessly in all environments. -->

Winden initially relied on a messy architecture that was hard to maintain. It used different setups for development and production‚ÄîPlay CDN for development and GitHub Actions for production‚Äîleading to inconsistencies and bugs. To simplify things, we moved to using Play CDN in both environments. However, this approach is still somewhat unreliable, as it depends on JSPM CDN to run the engine.

WindPress fixes this by building a new unified engine that works seamlessly in both development and production environments. The new core is lighter, faster, and built to last ‚Äî and yes, it‚Äôs fully open source. [View the engine on GitHub ‚Üí](https://github.com/wind-press/windpress/tree/main/assets/packages/core/tailwindcss)

Rebuilding the core package from scratch has allowed us to reduce the dependencies and streamline the codebase, making it easier to maintain and extend. This new architecture is designed to be more robust, efficient, and future-proof, ensuring that WindPress can grow and adapt as the web evolves. The dual Tailwind CSS support (both `3.x` and `4.x`) is a testament to this flexibility and adaptability of the new architecture.

### üìÅ Simple Virtual File System

WindPress introduces a virtual file system that acts like a file manager for your Tailwind CSS project. It helps you organize styles in a cleaner, more structured way by allowing you to split them into separate files and import them into a main stylesheet. ![](/assets/landing/feature-bento-simple-file-system.webp)

### üì¶ Bundled Assets for Full Offline Support

In line with [WordPress.org plugin guidelines](https://developer.wordpress.org/plugins/wordpress-org/common-issues/#:~:text=Files:%20Calling%20files%20remotely), WindPress includes all plugin's assets directly in the plugin ‚Äî served locally from your server.

You get:

* Offline-ready performance
* No reliance on external CDNs
* Faster and more private environments

However, an internet connection is still needed to load Tailwind CSS plugins from the `esm.sh` CDN.

### üß© Built-in Tailwind CSS Plugins Support

Use Tailwind CSS plugins like `@tailwindcss/forms`, `@tailwindcss/typography`, and more with zero setup. Just import them into your config, and WindPress handles the rest ‚Äî loading any `npm` packages via the `esm.sh` CDN automatically.
